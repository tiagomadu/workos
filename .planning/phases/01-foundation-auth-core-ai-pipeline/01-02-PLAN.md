**backend/app/services/transcript.py** — Transcript format detection and normalisation:

1. `detect_format(text: str) -> str` — Returns "macwhisper", "whisper_cli", or "plain".
   - MacWhisper pattern: lines matching `^\d{2}:\d{2}:\d{2}\s+.+?:` (timestamp followed by speaker label)
   - Whisper CLI pattern: lines matching `^\[\d{2}:\d{2}:\d{2}\.\d{3}\s*-->\s*\d{2}:\d{2}:\d{2}\.\d{3}\]`
   - Default to "plain" if neither pattern matches >30% of non-empty lines

2. `normalise_transcript(text: str) -> str` — Detects format and normalises:
   - MacWhisper: Strip leading timestamps (`HH:MM:SS`), keep `Speaker: text` format
   - Whisper CLI: Strip timestamp brackets `[HH:MM:SS.mmm --> HH:MM:SS.mmm]`, keep the text content. If no speaker labels exist, just concatenate text with newlines.
   - Plain: Return as-is (strip leading/trailing whitespace only)
   - Always normalize line endings to \n, strip trailing whitespace per line

3. `validate_transcript(text: str) -> tuple[bool, str]` — Returns (is_valid, error_message):
   - Reject empty or whitespace-only text: "Transcript is empty"
   - Reject text over 500KB (512000 bytes when encoded as UTF-8): "Transcript exceeds 500KB limit"
   - Accept everything else

**backend/app/services/storage.py** — Supabase Storage operations:

1. `upload_transcript(user_id: str, filename: str, content: bytes) -> str` — Uploads to the "transcripts" bucket at path `{user_id}/{YYYY}/{MM}/{YYYY-MM-DD}_{filename}`. Returns the storage path (NEVER a signed URL). Uses `supabase.storage.from_("transcripts").upload(path, content, {"content-type": "text/plain"})`.

2. `get_transcript_url(storage_path: str, expires_in: int = 900) -> str` — Generates a signed URL valid for 15 minutes. Uses `create_signed_url()`.

3. `create_meeting_record(user_id: str, transcript_path: str, raw_transcript: str, title: str | None, meeting_date: str | None) -> str` — Inserts into meetings table with status="pending". Returns the new meeting UUID. Uses the Supabase client with service role key (not user JWT) because background processing needs access.

4. `update_meeting_status(meeting_id: str, status: str, error_message: str | None = None) -> None` — Updates the status and optionally error_message. If status is "completed", also sets processed_at to NOW().

5. `get_meeting(meeting_id: str, user_id: str) -> dict | None` — Fetches a single meeting by ID, filtered by user_id for security. Returns None if not found.

**backend/app/api/v1/meetings.py** — Two endpoints plus a status endpoint:

1. `POST /api/v1/meetings/upload` (status_code=202) — Accepts `UploadFile` + optional `MeetingCreate` form fields. Validates: file must be .txt, file size <= 500KB (check via seek/tell pattern from Research Section 5). Reads content, decodes UTF-8, normalises with `normalise_transcript()`, uploads to storage, creates meeting record with status="pending", returns `MeetingUploadResponse`. Does NOT trigger background processing yet (Plan 03 wires that).

2. `POST /api/v1/meetings/paste` (status_code=202) — Accepts JSON body with `text: str` and optional MeetingCreate fields. Validates text via `validate_transcript()`. Normalises, uploads to storage (filename = "pasted_{timestamp}.txt"), creates meeting record, returns `MeetingUploadResponse`.

3. `GET /api/v1/meetings/{meeting_id}` — Returns full `MeetingResponse` for the given meeting. Uses `get_meeting()` scoped to the authenticated user. Returns 404 if not found.

Register the meetings router in `backend/app/main.py` with prefix `/api/v1`.
