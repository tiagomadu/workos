**CRITICAL: Database infrastructure already exists.** The `document_embeddings` table, HNSW index, and `match_documents()` function were created in Phase 1 migration. No new migrations needed.

**CRITICAL: Embedding model.** Use nomic-embed-text via Ollama's API at /api/embeddings endpoint. This runs ALWAYS through Ollama regardless of LLM_PROVIDER setting. The Ollama embeddings endpoint is:
```
POST http://localhost:11434/api/embeddings
{
  "model": "nomic-embed-text",
  "prompt": "text to embed"
}
Response: { "embedding": [0.1, 0.2, ...] }  # 768 dimensions
```

Use httpx.AsyncClient to call this endpoint directly (do NOT use the LLM Provider Protocol for embeddings — that's for structured text generation only).

**backend/app/services/embeddings.py** — Embedding generation service:

1. `async def get_embedding(text: str, ollama_url: str = None) -> list[float]`:
   - Call Ollama /api/embeddings with model="nomic-embed-text"
   - Use httpx.AsyncClient with 30s timeout
   - ollama_url defaults to settings.OLLAMA_BASE_URL (from config)
   - Return the 768-dim embedding vector
   - On error: log warning, return None

2. `def chunk_summary(summary: dict, meeting_id: str) -> list[dict]`:
   - Takes a MeetingSummary dict (overview, key_topics, decisions, follow_ups)
   - Creates one chunk per section:
     - chunk_text = f"Meeting Overview: {summary['overview']}"
     - chunk_text = f"Key Topics: {', '.join(summary['key_topics'])}"
     - chunk_text = f"Decisions Made: {', '.join(summary['decisions'])}"
     - chunk_text = f"Follow-ups: {', '.join(summary['follow_ups'])}"
   - Returns list of {chunk_text, chunk_index, metadata: {section, meeting_id}}

3. `def chunk_transcript(transcript: str, meeting_id: str, chunk_size: int = 500, overlap: int = 50) -> list[dict]`:
   - Split transcript into words
   - Create sliding window chunks of ~chunk_size words with overlap
   - Returns list of {chunk_text, chunk_index, metadata: {type: "transcript", meeting_id}}
   - chunk_index starts after summary chunks (e.g., 4, 5, 6, ...)

4. `async def generate_embeddings(meeting_id: str, user_id: str) -> int`:
   - Fetch meeting record (summary + raw_transcript)
   - Delete existing embeddings for this meeting (for reprocessing support)
   - Chunk summary (4 chunks) + chunk transcript (variable chunks)
   - For each chunk: generate embedding, insert into document_embeddings table
   - Insert with: user_id, meeting_id, chunk_index, chunk_text, embedding (as vector string), metadata (JSONB)
   - Return count of embeddings generated
   - On embedding error for a chunk: skip that chunk, log warning, continue

**backend/app/services/search.py** — Search service:

1. `async def search_meetings(query: str, user_id: str, date_from: str = None, date_to: str = None, meeting_type: str = None, limit: int = 10) -> dict`:
   - Step 1: Embed the query using get_embedding()
   - Step 2: Call match_documents() via Supabase RPC:
     ```python
     supabase = get_supabase_client()
     result = supabase.rpc("match_documents", {
         "query_embedding": embedding_str,  # format as string for pgvector
         "match_threshold": 0.7,
         "match_count": limit,
         "filter_user_id": user_id
     }).execute()
     ```
     Note: The match_documents function returns id, meeting_id, chunk_text, similarity.

   - Step 3: Enrich results with meeting metadata (title, date, type) by batch-fetching from meetings table

   - Step 4: Apply post-filters (date_from, date_to, meeting_type) on the enriched results
     - date_from: filter meeting_date >= date_from
     - date_to: filter meeting_date <= date_to
     - meeting_type: filter meeting_type == meeting_type

   - Step 5: Generate AI answer using top chunks as context
     - Render generate_answer.j2 prompt with query + top 5 chunk texts
     - Call active LLM provider (Ollama or Claude) for answer
     - Use a simple string response (not structured output — just text)

   - Return: { answer: str, sources: [{meeting_id, meeting_title, meeting_date, meeting_type, chunk_text, similarity}] }

**frontend/src/lib/api.ts** — Add search function:
```typescript
export async function searchMeetings(
  query: string, token: string, filters?: SearchFilters
): Promise<SearchResult> {
  const params = new URLSearchParams({ q: query })
  if (filters?.date_from) params.set('date_from', filters.date_from)
  if (filters?.date_to) params.set('date_to', filters.date_to)
  if (filters?.meeting_type) params.set('meeting_type', filters.meeting_type)
  const res = await fetch(`${API_URL}/api/v1/search?${params}`, {
    headers: { Authorization: `Bearer ${token}` },
  })
  if (!res.ok) {
    const err = await res.json().catch(() => ({}))
    throw new Error(err.detail || 'Search failed')
  }
  return res.json()
}
```
