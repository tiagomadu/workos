**CRITICAL: Database tables already exist.** The `action_items` table has: id, user_id, meeting_id, description, owner_name, owner_id, owner_status, due_date, status (not_started|in_progress|complete|cancelled), project_id, is_archived. The `projects` table has: id, user_id, name, description, status (on_track|at_risk|blocked|complete|archived), team_id. No new migrations needed.

**CRITICAL: Status mapping.** The DB has 4 statuses (not_started, in_progress, complete, cancelled) but the context decision says 3 UI statuses (To Do, In Progress, Done). Map: To Do = not_started, In Progress = in_progress, Done = complete. Keep cancelled in DB for archiving but don't expose as a primary status in the tracker UI. Similarly, projects use Active/Archived in UI but DB has more granular statuses -- map Active = any of (on_track, at_risk, blocked), Archived = archived.

class ProjectCreate(BaseModel):

class ProjectUpdate(BaseModel):
    name: Optional[str] = None

    team_id: Optional[str] = None

class ProjectResponse(BaseModel):
    id: str
    name: str
    description: Optional[str] = None
    status: str
    team_id: Optional[str] = None
    team_name: Optional[str] = None
    created_at: str

class ProjectDetailResponse(ProjectResponse):
    meeting_count: int = 0
    total_tasks: int = 0
    completed_tasks: int = 0
    overdue_tasks: int = 0
    meetings: list[dict] = []  # [{id, title, meeting_date, status}]

**backend/app/services/tasks.py** -- Task service functions:

1. `async def get_tasks(user_id, filters: TaskFilters) -> list[dict]` -- Query action_items for user_id. Apply filters:
   - status filter: WHERE status = filters.status
   - owner_id filter: WHERE owner_id = filters.owner_id
   - project_id filter: WHERE project_id = filters.project_id
   - archived filter: WHERE is_archived = False (default), or include if include_archived=True
   - Compute is_overdue: due_date < today AND status NOT IN ('complete', 'cancelled')
   - Sort: default overdue first, then by due_date ascending (nulls last)
   - Join with meetings for meeting_title, projects for project_name

2. `async def get_task(user_id, task_id) -> dict | None`

3. `async def create_task(user_id, data: TaskCreate) -> dict` -- Insert into action_items table. Set meeting_id=None for standalone tasks. Default due_date to 7 days from today if not provided. If owner_id provided, set owner_status='resolved'.

4. `async def update_task(user_id, task_id, data: TaskUpdate) -> dict` -- Update provided fields.

5. `async def archive_task(user_id, task_id) -> dict` -- Set is_archived=True.

6. `async def unarchive_task(user_id, task_id) -> dict` -- Set is_archived=False.

**backend/app/services/projects.py** -- Project service functions:

1. `async def get_projects(user_id, include_archived: bool = False) -> list[dict]` -- Query projects for user_id. Exclude status='archived' by default. Join with teams for team_name.

2. `async def get_project(user_id, project_id) -> dict | None` -- Full project detail with:
   - Count of linked meetings (meetings WHERE project_id = project_id)
   - Task stats: total, completed, overdue counts from action_items WHERE project_id = project_id
   - List of linked meetings (id, title, meeting_date, status)

3. `async def create_project(user_id, data: ProjectCreate) -> dict`

4. `async def update_project(user_id, project_id, data: ProjectUpdate) -> dict`

5. `async def archive_project(user_id, project_id) -> dict` -- Set status='archived'.
